## Decisions log (so they are not relitigated)
- **Official reference class = the synthetic field** — a recency-weighted topic-mixture blend of full (subfield, vintage) cohorts (reference-based → community migration). The cited-only co-citation *neighborhood* is **not** a ranking cohort (it's popularity-biased and has no uncited papers); it only informs the topic mixture. (QaL_spec §3/§5.)
- **Denominator rule:** every percentile is over the **full population** (cited + uncited), never cited-only and never a top-N/sorted sample; the uncited atom takes the **mid-rank 100·p₀/2**; the same convention is used for r_obs and r∞.
- Calibration seed = Decision Sciences subfields **1803, 1802, 1800** (extensions 1405, 1804 available).
- QaL = calibrated posterior over eventual percentile (point + 90% interval + bucket probs); back-tested to ~90% coverage via per-age split-conformal.
- **No single author-level score** (author page shows a per-paper distribution).
- Stack = GitHub Actions (**light** batch: calibration, synth worker, coverage) + **on-demand AWS EC2 us-east-1** (**heavy** full-corpus snapshot scan, `factory.py`) + Vercel/Next.js (web + read API) + Neon Postgres; **Vercel does no heavy compute**. The full-corpus scan runs in-region on a self-terminating EC2 box rather than Actions because Actions has a 6h cap and pays cross-cloud transfer from the AWS-hosted OpenAlex S3; in-region it's ~1h/~$0.50. The factory streams the snapshot to the CDF and never persists `works` (per `caching_architecture.md`). Note: this network blocks outbound 22 and 5432, so the box takes no SSH/Neon connections from here — it self-runs and reports via the serial console.
- **Synthetic-field construction is a fallback chain** (references → co-citation → content-and-authorship prior → single label), computed on the fly for any paper, so OpenAlex's single label is never the official reference class. (`docs/QaL_spec.md` §5.)
- **Layer-B conformal radius = cross-vintage (LOVO) at α=0.08.** A single fit/cal split under-covered held-out vintages; the cross-vintage radius fixes it and α=0.08 targets the honest ~90% actual coverage. Gate unchanged (0.88 ≤ LOVO coverage ≤ 0.97). A subfield that genuinely under-covers (e.g. Economics: two anomalous vintages) stays `fitted`/pending rather than being forced over — honesty over coverage. (`docs/calibration_decisions.md`.)
- **The single-field "stand-in" is the OpenAlex primary *subfield*, not the field.** When a paper's synthetic community isn't calibrated, the displayed forecast falls back to that paper's single OpenAlex **subfield** cohort (e.g. *Artificial Intelligence*, sid 1702) — which must itself be gate-passed for a number to show — NOT the broader OpenAlex **field** ("Computer Science", shown only in the evidence grid), and NOT the dominant *synthetic* component (which may be a different, uncalibrated subfield like Health Informatics). QaL calibrates at subfield granularity. The hero names the subfield so the basis is unambiguous.
- **Fitted-tier forecasts are shown, labeled "preliminary".** A synthetic forecast appears when ≥50% of the reference class is calibrated (gate-passed OR fitted); it's certified "gate-passed" only when ≥50% is back-tested, else shown as fitted/preliminary with the back-tested share. This relaxes *display* (more papers get a forecast) without relaxing the *gate* (the gate-passed badge still means ≥0.88 LOVO coverage). `official` stays the synthetic field only when it yields a forecast; an observed-only synthetic with a calibrated single-subfield keeps the subfield official (no downgrade to pending).
- **Explore is cache-only for the synthetic blend; the paper page computes it via a gated, cached endpoint.** Explore reads the cached `synthetic_field` table (one batched query, no per-row OpenAlex call) to keep page loads fast and off the API budget; coverage of the blend on Explore is a *prefill* question (currently the top ~2,000 by citations). The paper page serves **cache-only on the server GET** (universal layer + any cached synthetic); the synthetic field is computed only by a **client-triggered** `POST /api/synthetic/:oaid` — a behavior gate, run by a real browser after the page mounts — which caches to `synthetic_cache`. So flat crawler GETs never trigger the ~15-40-call compute. This **replaced the earlier hard index-gate** (the budget vector was the premium key, now retired — the web runs all-polite, so the synthetic's calls are free).
